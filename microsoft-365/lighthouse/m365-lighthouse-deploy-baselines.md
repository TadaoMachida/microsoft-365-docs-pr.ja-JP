# <a name="deploy-microsoft-365-lighthouse-baselines"></a>Microsoft 365 Lighthouseベースラインをデプロイする

Microsoft 365 Lighthouseでは、標準のマネージド テナント構成をデプロイして、顧客テナント内のユーザー、デバイス、データをセキュリティで保護できます。 Lighthouse には 7 つの [既定のベースライン構成](m365-lighthouse-deploy-standard-tenant-configurations-overview.md) があります。 Lighthouse 展開計画機能を使用すると、すべてのテナントにセキュリティ構成を表示、テスト、デプロイできます。 展開計画は、アクティブなテナントでのみ使用できます。 テナントがオンボードされたら、顧客の現在の構成を既定のベースライン構成と比較し、適切なアクションを実行できます。

## <a name="before-you-begin"></a>はじめに

お客様とテナントが、「[Microsoft 365 Lighthouseの要件」](m365-lighthouse-requirements.md)に記載されている要件を満たしていることを確認します。

## <a name="view-a-deployment-plan"></a>展開計画を表示する

1. Lighthouse の左側のナビゲーション ウィンドウで、[ **テナント**] を選択します。

2. テナントの一覧から、表示するテナントを選択します。

3. [ **展開計画]** タブを選択します。

    [展開計画] タブには、テナントの展開計画に含まれる各展開手順の検索可能なエクスポート可能な一覧が表示されます。これには、展開手順ごとに次の情報が含まれます。

    | 列            | 説明 |
    |-----------------|-------------------------------------------------------------------------------------|
    | デプロイ手順 | 展開手順の説明。                                                     |
    | 状態          | デプロイ 手順の状態。                                                  |
    | 基準        | デプロイ ステップの派生元となるベースライン。                             |
    | カテゴリ        | 展開手順がデバイス、ID、またはデータの管理に関連付けられているかどうか。 |
    | 最終更新日時    | デプロイ ステップが最後に更新された日付。                             |

4. 一覧から、確認する展開手順を選択します。

    [展開手順] ページには、次の情報が表示されます。

    | 列            | 説明 |
    |-------------------|-----------------------------------------------------------------------------------------------|
    | 概要        | 展開手順の目的の概要。                                         |
    | 基準       | デプロイ ステップの派生元となるベースライン。                             |
    | カテゴリ       | 展開手順がデバイス、ID、またはデータの管理に関連付けられているかどうか。 |
    | 必要な SKU   | デプロイ手順を完了するために必要な SKU。                                      |
    | ユーザーへの影響    | テナントのユーザーにステップをデプロイした場合の影響。                             |
    | ユーザー向け | テナントのユーザーが役に立つリソースへのリンク。                             |
    | 次の手順     | 該当する次の手順に関するリンクとガイダンス。                                |

    デプロイ手順には、完了する必要がある 1 つ以上のプロセスが含まれます。 [デプロイ ステップ] ページには、デプロイ 手順に含まれる各プロセスを一覧表示し、次の情報を示すテーブルが含まれています。

    | 列            | 説明 |
    |-------------------|-------------------------------------------------------------|
    | プロセス名      | プロセスの名前を指定すると、該当する [プロセス] タブが開きます。          |
    | 状態            | 展開プロセスに含まれるこれらの設定構成の状態を検出しました。           |
    | 管理ポータル | プロセスに関連付けられている構成設定を管理するポータル。 |

## <a name="deploy-a-deployment-step"></a>デプロイ手順をデプロイする

1. 左側のナビゲーション ページで、[テナント] を選択 **します**。

2. テナントの一覧から、表示するテナントを選択します。

3. [ **展開計画]** タブを選択します。

4. 展開手順の一覧から、デプロイするデプロイ ステップを選択します。

5. [ **確認とデプロイ] を選択します**。

6. **[構成の確認**] ウィンドウで、[**デプロイ**] を選択します。

## <a name="test-a-deployment-step"></a>デプロイ 手順をテストする

条件付きアクセス ポリシーを通じて展開される展開手順では、テナントに設定を展開せずに、展開手順の構成設定と既存のポリシーの設定を比較できます。

1. 左側のナビゲーション ページで、[テナント] を選択 **します**。

2. テナントの一覧から、表示するテナントを選択します。

3. [ **展開計画]** タブを選択します。

4. 展開手順の一覧から、デプロイするデプロイ ステップを選択します。

5. [ **確認とデプロイ] を選択します**。

6. **[構成の確認]** ウィンドウで、**展開なしでこれらの設定をテストするを選択します**。

7. [ **テスト**] を選択します。

[構成の確認] ウィンドウが閉じ、ポリシーの比較が表示されます。 既存のテナント内の各ポリシーは、[検出された設定] テーブルに一覧表示されます。

[検出された設定] テーブルには、既存の各ポリシーが一覧表示され、設定の数と、次のいずれかの状態のユーザーの数がかっこで囲まれてまとめられます。

| 状態         | 説明
|-------------|------------------------------------------------------------|
| 等しい設定       | テナント内の同等の値を持つ展開計画の構成設定の合計数。      |
| 不足している設定     | テナント内の値が不足している展開計画の構成設定の合計数。      |
| 競合する設定 | テナント内で競合する値を持つ展開計画の構成設定の合計数。 |

検出された設定は、設定とユーザー レベルで各ポリシーの構成設定の詳細を提供し、次の各設定の状態で並べ替えることができるモジュールテーブルで表示することもできます。

| 状態         | 説明
|-------------|------------------------------------------------------------|
| 合計設定       | デプロイ プロセスに含まれる構成設定の合計数。                        |
| 等しい設定       | テナント内の同等の値を持つ展開計画の構成設定の合計数。      |
| 不足している設定     | テナント内の値が不足している展開計画の構成設定の合計数。      |
| 競合する設定 | テナント内で競合する値を持つ展開計画の構成設定の合計数。 |
| 追加の設定       | テナント内の値を持ち、展開計画に値がない構成設定の合計数。     |

この比較が行われると、Lighthouse は検出された状態、デプロイの状態、デプロイ ステップの状態を自動的に更新します。

比較する既存のポリシーがない場合は、[確認と展開] を選択して [構成の確認] ウィンドウを再度開き、[デプロイ] を選択します。

比較する既存のポリシーがある場合は、次のいずれかを実行できます。

- 展開計画の構成設定を編集し、既存のポリシーに対して再テストし、[ **確認と展開** ] を選択して [構成の確認] ウィンドウを再度開き、目的の構成設定を調整し、チェック ボックスを再選択して、ウィンドウの下部にある **[テスト** ] を選択します。

- 該当する管理ポータル内の既存のポリシーを編集して、次のいずれかの方法で違いを調整します。
  - 不足している設定の適用
  - 競合する設定の編集
  - 既存のポリシーの削除

Lighthouse を使用して自動化できるデプロイ プロセスごとに、デプロイの状態と検出された状態の両方があります。

- 検出された状態は、このプロセスの設定が現在デプロイされている範囲を示します。
- デプロイの状態は、テナントへの最後のデプロイの状態です。

展開手順は、既存のポリシーに関係なくデプロイできますが、競合する設定がなくなるまで完了とは見なされません。 これらの競合する設定を解決できないと、ユーザー エクスペリエンスに影響を与える可能性があります。 

既存のポリシーからテナントに等しい設定が存在する場合、デプロイ ステップをインスタンスにデプロイすると、テナント内の既存の設定が重複しますが、ユーザー エクスペリエンスには影響しません。 

意識を高めるために追加の設定が提供されますが、アクションを実行する必要はありません。

ポリシー競合管理の詳細については、 [Azure AD の条件付きアクセスに関するドキュメントを参照してください](/azure/active-directory/conditional-access/)。

## <a name="update-deployment-step-status"></a>デプロイ ステップの状態を更新する

1. 左側のナビゲーション ページで、[テナント] を選択 **します**。

2. テナントの一覧から、表示するテナントを選択します。

3. [ **展開計画]** タブを選択します。

4. 展開手順の一覧から、更新する展開手順を選択します。

5. [ **宛先アドレス** ] ドロップダウン リストから、アクションの状態を選択します。

    | アクションの状態                        | 説明      |
    |---------------------------------------|----------------------------------------|
    | アドレスを指定するには                        | 複数のデプロイ ステップ プロセスを含まないすべてのデプロイ ステップの既定の状態。      |
    | 計画                           | デプロイ手順は計画済みですが、まだ完了していません。                                      |
    | 受け入れられたリスク                     | ユーザーは、展開手順を適用することで回避される可能性があるリスクを受け入れています。 |
    | サード パーティを通じて解決されるリスク | リスクは、サード パーティ製のアプリケーションまたはソフトウェアの実装によって解決されました。             |
    | 代替手段によって解決される  | リスクは、内部ツールの実装など、代替手段によって解決されています。    |
    | 手動構成の適用      | 展開計画に規定されている構成は、手動で適用されています。                         |

## <a name="share-deployment-step"></a>展開の共有手順

1. 左側のナビゲーション ページで、[テナント] を選択 **します**。

2. テナントの一覧から、表示するテナントを選択します。

3. [ **展開計画]** タブを選択します。

4. 展開手順の一覧から、共有するデプロイ ステップを選択します。

5. **[共有**] ドロップダウン リストから、次のいずれかのオプションを選択します。

    | オプション  | 説明 |
    |-----------|-------------------------------------------------------------------------|
    | Copy  | デプロイ ステップへのリンクをクリップボードにコピーします。                                     |
    | メール | ローカル コンピューターで新しい電子メール メッセージを開き、展開手順へのリンクを挿入します。 |

    このリンクを使用すると、組織内のアクセス許可を持つすべてのユーザーがテナントの展開計画を表示できるようになります。
