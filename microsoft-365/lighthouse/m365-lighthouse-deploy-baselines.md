# <a name="deploy-microsoft-365-lighthouse-baselines"></a>ベースラインMicrosoft 365 Lighthouse展開する

Microsoft 365 Lighthouseユーザー、デバイス、およびデータを顧客テナント内でセキュリティで保護するために、標準の管理テナント構成を展開できます。 ライトハウスに標準 [で提供される 7](m365-lighthouse-deploy-standard-tenant-configurations-overview.md) つの既定のベースライン構成があります。 ライトハウス展開計画機能を使用すると、すべてのテナントでセキュリティ構成を表示、テスト、展開できます。 展開計画は、アクティブなテナントでのみ使用できます。 テナントがオンボードされた後、顧客の現在の構成を既定のベースライン構成と比較し、適切なアクションを実行できます。

## <a name="before-you-begin"></a>はじめに

ユーザーと顧客テナントが「要件」に記載されている要件を満[たしていることを確認](m365-lighthouse-requirements.md)Microsoft 365 Lighthouse。

## <a name="view-a-deployment-plan"></a>展開計画の表示

1. 左側のナビゲーション ページで、[テナント] **を選択します**。

2. テナントの一覧から、表示するテナントを選択します。

3. [展開計画 **] タブを選択** します。

    [展開計画] タブには、テナントの展開計画に含まれる各展開手順の検索およびエクスポート可能なリストが表示され、各展開手順に関する次の情報が含まれます。

    | 列            | 説明 |
    |-----------------|-------------------------------------------------------------------------------------|
    | 展開手順 | 展開手順の説明。                                                     |
    | 状態          | 展開手順の状態。                                                  |
    | 基準        | 展開手順の派生基準。                             |
    | カテゴリ        | 展開手順がデバイス、ID、またはデータの管理に関連付けられているかどうか。 |
    | 最終更新日時    | 展開手順が最後に更新された日付。                             |

4. 一覧から、確認する展開手順を選択します。

    [展開手順] ページには、次の情報が表示されます。

    | 列            | 説明 |
    |-------------------|-----------------------------------------------------------------------------------------------|
    | 概要        | 展開手順の目的の概要。                                         |
    | 基準       | 展開手順の派生基準。                             |
    | カテゴリ       | 展開手順がデバイス、ID、またはデータの管理に関連付けられているかどうか。 |
    | 必須の SKU   | 展開手順を完了するために必要な SKU。                                      |
    | ユーザーへの影響    | テナントのユーザーにステップを展開する場合の影響。                             |
    | ユーザーの場合 | テナントのユーザーが役立つリソースへのリンク。                             |
    | 次の手順     | 該当する次の手順に関するリンクとガイダンス。                                |

    展開手順は、展開手順の要件を満たすために完了する必要がある 1 つ以上のプロセスで構成されます。 [展開手順] ページには、展開手順に含まれる各プロセスの一覧と、次の情報を示すプロセス テーブルが含まれています。

    | 列            | 説明 |
    |-------------------|-------------------------------------------------------------|
    | プロセス名      | プロセスの名前を選択すると、該当する [プロセス] タブが開きます。          |
    | 状態            | 展開プロセスに含まれるこれらの設定構成の状態が検出されました。           |
    | 管理ポータル | プロセスに関連付けられた構成設定を管理するポータル。 |

## <a name="deploy-a-deployment-step"></a>展開手順の展開

1. 左側のナビゲーション ページで、[テナント] **を選択します**。

2. テナントの一覧から、表示するテナントを選択します。

3. [展開計画 **] タブを選択** します。

4. [展開手順] ボックスの一覧から、展開する展開手順を選択します。

5. [レビュー **と展開] を選択します**。

6. [構成の **確認] ウィンドウで、[** 展開] を **選択します**。

## <a name="test-a-deployment-step"></a>展開手順のテスト

条件付きアクセス ポリシーを使用して展開する展開手順では、テナントに設定を展開せずに、展開手順の構成設定と既存のポリシーの設定を比較できます。

1. 左側のナビゲーション ページで、[テナント] **を選択します**。

2. テナントの一覧から、表示するテナントを選択します。

3. [展開計画 **] タブを選択** します。

4. [展開手順] ボックスの一覧から、展開する展開手順を選択します。

5. [レビュー **と展開] を選択します**。

6. [構成 **の確認] ウィンドウで** 、[展開なしで **これらの設定をテストする] を選択します**。

7. [テスト **] を選択します**。

[構成の確認] ウィンドウが閉じ、ポリシーの比較が表示されます。 既存のテナント内の各ポリシーは、[検出された設定] テーブルに一覧表示されます。

[検出された設定] テーブルには、既存の各ポリシーが一覧表示され、設定の数と、次のいずれかの状態にあるユーザーの数をかっこで囲んで要約します。

| 状態         | 説明
|-------------|------------------------------------------------------------|
| 等しい設定       | テナント内の同等の値を持つ展開計画の構成設定の総数。      |
| 不足している設定     | テナントの値が不足している展開計画の構成設定の総数。      |
| 競合する設定 | テナント内の競合する値を持つ展開計画の構成設定の総数。 |

検出された設定は、設定とユーザー レベルで各ポリシーの構成設定の詳細を提供するモジュール式のテーブルで表示することもできます。また、次の設定の各状態で並べ替えできます。

| 状態         | 説明
|-------------|------------------------------------------------------------|
| 合計設定       | 展開プロセスに含まれる構成設定の総数。                        |
| 等しい設定       | テナント内の同等の値を持つ展開計画の構成設定の総数。      |
| 不足している設定     | テナントの値が不足している展開計画の構成設定の総数。      |
| 競合する設定 | テナント内の競合する値を持つ展開計画の構成設定の総数。 |
| 追加の設定       | テナント内の値を持つ構成設定の総数ですが、展開計画に値はありません。     |

この比較が行われた場合、ライトハウスは検出済み状態、展開状態、および展開ステップの状態を自動的に更新します。

比較する既存のポリシーがない場合は、[レビューと展開] を選択して [構成の確認] ウィンドウを再度開き、[展開] を選択します。

比較する既存のポリシーがある場合は、次のいずれかを実行できます。

- 展開計画の構成設定を編集し、既存のポリシーに対して再テストし、[確認と展開] を選択して [構成の確認] ウィンドウを再度開き、目的の構成設定を調整し、チェック ボックスをオンにし、ウィンドウの下部にある [テスト] を選択します。

- 該当する管理ポータル内の既存のポリシーを編集して、次のいずれかの方法で相違点を調整します。
  - 不足している設定の適用
  - 競合する設定の編集
  - 既存のポリシーの削除

ライトハウス経由で自動化できる展開プロセスごとに、展開状態と検出された状態の両方があります。

- 検出された状態は、このプロセスの設定が現在展開されている範囲を示します。
- 展開状態は、テナントへの最後の展開の状態です。

展開手順は、既存のポリシーに関係なく展開できますが、競合する設定がない限り完全とは見なされません。 これらの競合する設定を解決できないと、ユーザー エクスペリエンスに影響を与える可能性があります。 

既存のポリシーからテナントに等しい設定が存在する場合のインスタンスでの展開ステップの展開は、テナント内の既存の設定の重複を生じしますが、ユーザー エクスペリエンスには影響を与えかねない。 

追加の設定は、ユーザーの認識のために提供されますが、アクションを実行する必要はありません。

ポリシーの競合管理の詳細については、「条件付き[アクセスAzure AD参照してください](/azure/active-directory/conditional-access/)。

## <a name="update-deployment-step-status"></a>展開手順の状態を更新する

1. 左側のナビゲーション ページで、[テナント] **を選択します**。

2. テナントの一覧から、表示するテナントを選択します。

3. [展開計画 **] タブを選択** します。

4. 展開手順の一覧から、更新する展開手順を選択します。

5. [To **address]** ドロップダウン リストから、アクションの状態を選択します。

    | アクションの状態                        | 説明      |
    |---------------------------------------|----------------------------------------|
    | アドレス指定                        | 複数の展開手順プロセスを含めないすべての展開手順の既定の状態。      |
    | 計画済み                           | 展開手順は計画済みですが、まだ完了していません。                                      |
    | リスクの受け入れ                     | ユーザーは、展開手順を適用することによって回避される可能性があるリスクを受け入れ、 |
    | 第三者によるリスク解決 | リスクは、サード パーティ製のアプリケーションまたはソフトウェアの実装によって解決されています。             |
    | 代替手段によって解決される  | リスクは、内部ツールの実装など、代替手段によって解決されています。    |
    | 手動構成の適用      | 展開計画に規定されている構成が手動で適用されています。                         |

## <a name="share-deployment-step"></a>展開の共有手順

1. 左側のナビゲーション ページで、[テナント] **を選択します**。

2. テナントの一覧から、表示するテナントを選択します。

3. [展開計画 **] タブを選択** します。

4. [展開手順] リストから、共有する展開手順を選択します。

5. [共有 **] ドロップダウン** リストから、次のいずれかのオプションを選択します。

    | オプション  | 説明 |
    |-----------|-------------------------------------------------------------------------|
    | コピー  | 展開手順へのリンクをクリップボードにコピーします。                                     |
    | メール | ローカル コンピューターで新しい電子メール メッセージを開き、展開手順へのリンクを挿入します。 |

    このリンクを使用すると、組織内のアクセス許可を持つすべてのユーザーがテナントの展開計画を表示できます。
