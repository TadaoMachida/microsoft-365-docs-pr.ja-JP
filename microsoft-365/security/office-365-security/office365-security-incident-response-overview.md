 **概要：** このソリューションは、Office 365における最も一般的なサイバーセキュリティ攻撃の指標、特定の攻撃を積極的に確認する方法、およびそれに対応する方法について説明します。

## <a name="learn-how-to-respond-to-cyberattacks"></a>サイバー攻撃に対応する方法について説明します

すべてのサイバー攻撃を阻止できるわけではありません。 攻撃者は常に防御戦略の新しい弱点を探しているか、古いものを悪用しています。 攻撃を認識する方法を知ることで、迅速に対応できるため、セキュリティ インシデントの期間が短縮されます。

この一連の記事は、特定の種類の攻撃がMicrosoft 365でどのようなものになるかを理解するのに役立ち、対応するために実行できる手順を示します。 これらは、次の点を理解するための簡単なエントリ ポイントです。

- 攻撃とは何か、どのように動作するか。

- 侵害のインジケーター (IOC) と呼ばれる、探す兆候とその検索方法。

- 攻撃を肯定的に確認する方法。

- 攻撃を遮断し、将来組織をより適切に保護するための手順。

- 各攻撃の種類に関する詳細な情報へのリンク。

より多くの記事が時間の経過と共に追加されるため、毎月ここに戻って確認してください。

## <a name="secure-microsoft-365-like-a-cybersecurity-pro"></a>サイバー セキュリティの専門家のように、Microsoft 365 のセキュリティを強化する

Microsoft 365 サブスクリプションには、データとユーザーを保護するために使用できる強力なセキュリティ機能のセットが用意されています。  [Microsoft 365セキュリティ ロードマップ - 最初の 30 日間、90 日間、およびそれ以降の最優先事項](security-roadmap.md)を使用して、Microsoft 365組織をセキュリティで保護するための Microsoft 推奨のベスト プラクティスを実装します。

- 最初の 30 日間で完了すべき作業。すぐにユーザーに影響しますが、それほど大きな影響ではありません。

- 90 日以内に実行するタスク。 これらは、計画と実装にもう少し時間がかかりますが、セキュリティ体制を大幅に改善します

- 90 日以降。最初の 90 日間の作業で保護が強化されます。
