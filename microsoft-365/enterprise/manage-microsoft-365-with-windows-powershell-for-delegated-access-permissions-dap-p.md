# <a name="how-to-manage-microsoft-365-with-windows-powershell-for-delegated-access-permissions-partners"></a>委任されたアクセス許可パートナーのWindows PowerShellを使用してMicrosoft 365を管理する方法

*この記事は、Microsoft 365 Enterprise および Office 365 Enterprise の両方に適用されます。*

委任アクセス許可 (DAP) パートナー とは、シンジケート パートナーとクラウド ソリューション プロバイダー (CSP) パートナーです。 多くはネットワークプロバイダーまたは通信プロバイダーです。 Microsoft 365サブスクリプションをサービス オファリングにバンドルします。 Microsoft 365 サブスクリプションを販売すると、顧客のテナンシーに対する代理管理 (AOBO) アクセス許可が自動的に付与され、それらのテナントを管理およびレポートできるようになります。 これらのタスクは、Microsoft 365 管理センターでは困難です。 PowerShell を使用すると、次のような管理タスクを実行Microsoft 365の方がはるかに簡単です。
- すべての顧客 **TenantId とそのドメインを** 一覧表示する 
- 顧客テナント内のすべてのユーザーとその割り当てられたライセンスを識別する
> [!NOTE]
> 一部の管理タスクは PowerShell でのみ実行できます。
