# <a name="microsoft-compliance-manager-alerts-and-alert-policies"></a>Microsoft コンプライアンス マネージャーのアラートとアラート ポリシー

**この記事では、次の情報を参照してください。** コンプライアンス マネージャーで **特定の** アクティビティのアラートを設定する方法、アラートを管理する方法、アラート条件を定義するためのアラート ポリシーを作成する方法について説明します。

## <a name="overview"></a>概要
コンプライアンス 管理人は、変更が発生するとすぐに通知を受け取り、コンプライアンスの目標を追跡できます。 たとえば、テナントの構成変更によって改善アクションのスコア値が増加または減少した場合、または実装またはテスト作業を実行するためにユーザーに改善アクションが割り当てられている場合に通知するアラートを設定できます。 アラートを [作成できるイベント](#create-an-alert-policy) の種類を表示します。

アラートを作成するには、まずアラート ポリシーを設定して、アラートをトリガーする条件と通知の頻度を概説します。 ポリシー条件との一致が検出されると、詳細が記載された電子メール通知が送信され、調査するか、さらにアクションを実行するか判断できます。


すべてのアラートがコンプライアンス 管理の [アラート] タブに表示され、すべてのアラート ポリシーが [アラート ポリシー] タブ **に表示されます**。

## <a name="understanding-the-alerts-and-alert-policies-pages"></a>[アラートとアラート のポリシー] ページについて

> [!IMPORTANT]
> コンプライアンス マネージャーの [**通知** と通知ポリシー] ページにアクセスするには、Azure Active Directory (AD) でセキュリティ リーダーの役割を保持する必要があります。 アラートとアラート ポリシーを使用するには、追加のセキュリティとコンプライアンス マネージャーの役割が必要です。 アラート ポリシーのアクセス許可 [の詳細については、以下をご覧ください](#alert-policy-permissions)。

### <a name="alert-policies-page"></a>[アラート ポリシー] ページ

コンプライアンス 管理 **の [アラート ポリシー]** タブを選択して、アラート ポリシーを表示および管理します。 [ **アラート ポリシー] ページ** には、組織によって作成されたポリシーの一覧を示すテーブルが表示されます。 このページから、新しいポリシーを作成し、既存のポリシーを編集し、ライセンス認証の状態を変更し、ポリシーを削除できます。

[状態 **] 列の** **[アクティブ** ] は、ポリシーが有効であり、条件が満たされるとアラートをトリガーします。 **非アクティブ** とは、ポリシーが存在するが、アラートを生成しない状態を意味します。 ポリシーテーブルには、ポリシーの重大度と、ポリシーが最後に変更された日付も表示されます。

個々のポリシーの詳細を表示するには、表の行を選択します。 すべての詳細を表示するフライアウト ウィンドウが表示されます。 ウィンドウの **下部にある** [アクション] ボタンを選択し、オプションから選択してポリシーの編集、アラートの表示、または削除を行います。 追加、編集、削除、アクティブ化、無効化を行うコマンドは、フィルターの上の表の上部付近でも使用できます。

アラート ポリシーの作成を開始するには、「アラート ポリシーの [作成」を参照してください](#create-an-alert-policy)。

### <a name="alerts-page"></a>[アラート] ページ

コンプライアンス マネージャーの **[アラート** ] タブを選択して、アラートを表示および管理します。 [ **アラート** ] ページには、アラート ポリシーによって生成された各アラートと、その重大度とトリガー イベント (アクションのスコアの変更など) とアラートの日付を示すテーブルが表示されます。

個々のアラートを表示するには、テーブル内の行を選択します。 ウィンドウの [概要] タブに、すべての詳細を表示する **フライ** アウト ウィンドウが表示されます。 [ **イベント ログ] タブ** には、アラートをトリガーしたユーザーが実行したアクションが表示されます。

ウィンドウ **の下部にある** [アクション] ボタンには、ユーザーに通知を割り当て、フォローアップを行うオプション、アラートを生成したアクションを実行したユーザーにメールを送信する、またはアラートを生成するポリシーの詳細を表示するオプションがあります。 また、行にカーソルを合わせるとアラート名の左側に表示される丸いボタンを選択し、フィルターの上にある表の上部にあるボタンのいずれかを選択することで、同じ操作を実行することもできます。

アラートの操作を開始するには、「アラートの表示 [と管理」を参照してください](#viewing-and-managing-alerts)。

## <a name="create-an-alert-policy"></a>アラート ポリシーを作成する

改善アクションに関連する特定の変更やイベントが発生した場合に警告するポリシーを作成できます。 イベントの種類を以下に示します。

### <a name="alert-event-types"></a>アラート イベントの種類

- **スコアの** 変更: 組織内の誰かが行った構成の変更により、改善アクションに対して与えるポイントの増減。 たとえば、組織がインサイダー リスク管理ポリシーを作成した場合、特定のアクションに対するポイントが一定の量増加する可能性があります。
- **割り** 当ての変更: 改善アクションがユーザーに割り当てられているか、別のユーザーに再割り当てされている、またはユーザーから割り当てられていない。
- **実装状態の変更**: ユーザーが改善アクションの実装状態を変更しました。
- **テストの状態の変更**: ユーザーが改善アクションのテスト状態を変更しました。
- **証拠の** 変更: ユーザーが、改善アクションの [ドキュメント] タブで証拠ドキュメントをアップロードまたは削除しました。

### <a name="policy-creation-steps"></a>ポリシーの作成手順

1 つ以上のイベントに基づいてアラートを生成するポリシーを作成するには、次の手順に従います。

1. コンプライアンス **マネージャーで、[** アラート ポリシー] ページ **に移動** し、[+ **追加** ] を選択してポリシー作成ウィザードを開始します。

2. [名前と **説明] ページ** で、ポリシーの名前とオプションの説明を入力し、[次へ] を選択 **します**。

3. [条件 **] ページ** で、アラートをトリガーする 1 つ以上のイベントを選択します。 [改善アクション **アクティビティ] ヘッダーで**、[サブ条件の追加] を選択し、各条件名の左側にホバーすると表示されるボックスをオンにします。 ポリシーの 1 つ以上の条件を選択できます。割り当ての変更、証拠の変更、実装状態の変更、スコアの変更、テストの状態の変更。 完了したら、[次へ] を **選択します**。

4. [結果 **] ページで** 、ポリシーの一致が検出された場合の対応を選択します。
      - 一致が検出された場合は、アラートの重大度レベル (低、中、高) を選択します。
      - 一致が検出された場合に電子メールで通知する頻度を選択します。 一致ごとに通知を受け取るか、3 つ以上の一致数のしきい値を選択できます。
      - 3 つ以上の一致後に通知を受け取る場合は、そのしきい値に達する必要がある分数を指定します (たとえば、90 分以内に 4 回の一致)。
  
    完了したら、**[次へ]** を選択します。

5. [通知 **の受信者] ページ** で、ポリシー条件が満たされた場合に電子メールを受信する組織の他のユーザーを選択します。 ポリシーを作成するユーザーが既定の受信者です。 [ **+受信者の選択]** を選択し、電子メール通知を受信するフライアウト ウィンドウの各ユーザー名の横にあるボックスをオンにします。 完了したら、[受信者の追加 **] を選択し**、[次へ] を **選択します**。

6. すべての選択内容を確認し、選択して各セクションに変更を加え、[次へ] を選択 **します**。 確認が完了したら、[ポリシーの作成] **を選択します**。

7. ポリシーが作成されると、[完了] を **選択します**。 作成したポリシー **のフライアウト** ウィンドウが開いている [アラート ポリシー] ページに到着します。

ポリシーを作成すると、ポリシーがアクティブになります。つまり、一致の検出とアラートの生成が開始されます。 ポリシーを **無効または削除する** 方法については、以下の「ポリシーの管理」セクションを参照してください。

ポリシーによってアラートが生成される前に、ポリシーを作成または更新してから最大 24 時間かかる場合があります。 イベントの [トリガーとアラート集約の](#view-alert-details) 詳細については、以下の「アラートの詳細を表示する」を参照してください。

## <a name="managing-policies"></a>ポリシーの管理

[ **アラート ポリシー] ページ** には、すべてのポリシーの一覧が表示されます。 このページ [の詳細については、「アラート ポリシー」](#alert-policies-page) ページを参照してください。 組織内のすべてのユーザーはポリシーを表示できますが、特定のアクションは特定の役割に制限されます。「 [アラート ポリシーのアクセス許可」を参照してください](#alert-policy-permissions)。

### <a name="view-policy-details"></a>ポリシーの詳細を表示する

[アラート ポリシー] ページの行からポリシーを選択し、ポリシーの詳細 (一致条件、アラート通知が送信されるかどうか、いつ、誰に送信するか、重大度レベルなど) を表示するフライアウト パネルを表示します。

パネル **の** 下部にある [アクション] ボタンには、ポリシーの編集、ポリシーの削除、アラートの表示を行うオプションが表示されます。

### <a name="view-a-policys-alerts"></a>ポリシーの通知を表示する

ポリシーのフライアウト パネルで、[アクション] を選択 **し** 、[アラートの表示 **] を選択します**。 そのポリシーによって生成されたアラートのフィルター処理されたビューを使用して、[アラート] ページに直接移動します。 アラートを使用 [する方法について学習します](#viewing-and-managing-alerts)。

### <a name="edit-a-policy"></a>ポリシーを編集する

ポリシーの名前以外の任意の側面を編集できます。 名前を変更する場合は、新しい名前で新しいポリシーを作成する必要があります。

ポリシーを編集するには、[アラート ポリシー] ページの行にカーソルを合わせ、フィルターの上の上部にある [編集]  ボタンを選択すると、その名前の左側に表示される丸いボタンを選択します。

ポリシー作成ウィザードが表示され、ポリシーに変更を加え、保存できます。  ポリシーを選択して詳細パネルを表示し、[アクション] ボタンから [ポリシーの編集] **を選択します**。 ウィザードを再度実行したら、選択内容を確認し、最後の手順で [ **更新** ] を選択して変更を保存します。

更新されたポリシーによってアラートが生成されるには、最大 24 時間かかる場合があります。

### <a name="activate-or-inactivate-a-policy"></a>ポリシーのアクティブ化または非アクティブ化

ポリシーは、作成されるとすぐに既定でアクティブ化されます。 アクティブな場合、ポリシーは条件が満たされた場合にアラート ([アラート] ページに表示) を作成し、指定された受信者に通知メールを送信します。

ポリシーを非アクティブな状態に変更するには、アラートを生成しません。行にカーソルを合わせると、ポリシー名の左側に表示される丸いボタンを選択します。 次に、表の **上にある [無効** にする] コマンドを選択します。 ポリシーの状態が [非アクティブ] と表示されます。 ポリシーを再アクティブ化するには、同じプロセスに従い、フィルターの上にある **[アクティブ化** ] ボタンを選択します。

### <a name="delete-a-policy"></a>ポリシーを削除する

ポリシーを削除するには、[アラート ポリシー] ページの名前の横にあるボタンを選択し、ページの上部にある [削除] を選択します。 ポリシーを選択して詳細パネルを表示し、[アクション] ボタンから [ポリシーの削除] **を選択します**。

削除は永続的です。 ポリシーを削除すると、通知や電子メール通知は生成されなくなりました。 削除されたポリシー [に接続されているアラートについて詳しくは、次のページをご覧ください](#when-policies-are-deleted)。

## <a name="viewing-and-managing-alerts"></a>アラートの表示と管理

[ **アラート]** ページには、すべてのポリシーによって生成されたすべてのアラートを含むテーブルが表示されます。 ポリシーの条件に一致するイベントが発生した直後にアラートが生成されます。 アラート名は、アラートを生成したポリシーと同じ名前です。

アラートは、アクティブ なポリシーからのみ生成できます。 アラートが生成されると、ポリシーがアクティブか非アクティブかにかかわらず、[アラート] ページに表示されたままになります。

### <a name="filter-your-view-of-alerts"></a>アラートのビューをフィルター処理する
[アラート] ページの表の上にある **[** フィルター] コマンドを選択すると、アラートのビューを **フィルター処理** できます。 [フィルター] **フライアウト** ウィンドウで、次のフィルター オプションの中から選択します。

- イベントの種類
- 重要度
- 状態
- 割り当てられたユーザー
- 検出日
- ポリシー名

選択を行った後、[適用] を **選択します**。 フライアウト ウィンドウが閉じ、更新された **[通知** ] ページにフィルター処理されたビューが表示されます。 フィルターはテーブルの上部に表示されます。一部のフィルター列がテーブルに表示される場合はありません。

### <a name="view-alert-details"></a>アラートの詳細を表示する

アラートをトリガーしたイベントを含む、アラートに関する詳細を表示するには、表の行を選択します。 フライアウト ウィンドウには、パネルの [概要] タブにアラート **の** 詳細が表示されます。

フライ **アウト パネルの** [イベント ログ] タブには、スコアの変更や割り当ての変更など、アラートを生成したアクティビティと、各アクションに関連付けられたユーザーの名前と検出された日付が一覧表示されます。

### <a name="alert-events"></a>アラート イベント

[ **アラート]** ページの **[イベント] 列** は、検出されたポリシーの条件を示します。つまり、アラートを生成したアクティビティです。 アラート **の詳細パネル** の [イベント ログ] タブには、イベントの各インスタンス、関連付けられたユーザー、および検出された日付が一覧表示されます。 イベントの値を以下に示します。

- **スコアの変更**: ポイントの増減数を示します
- **割り** 当ての変更: 改善アクションがユーザーに割り当てられているか、別のユーザーに再割り当てされている、またはユーザーから割り当てられていない
- **実装状態の変更**: ユーザーが改善アクションの実装状態を変更しました
- **テストの状態の変更**: ユーザーが改善アクションのテスト状態を変更しました。
- **証拠の** 変更: ユーザーが改善アクションの [ドキュメント] タブで証拠ドキュメントをアップロードまたは削除しました
- **複数イベント**: 同じ種類のイベントの複数のインスタンスが検出されました。たとえば、複数回再割り当てされた 1 つの改善アクション
- **複数の条件**: 1 つのポリシー内の複数の条件が検出されました

#### <a name="alert-aggregation-for-multiple-events-within-one-minute"></a>1 分以内に複数のイベントに対するアラート集約

アラート ポリシーの条件に一致する複数のイベントが 1 分で発生すると、アラート集約と呼ばれるプロセスによって既存のアラートに追加されます。

たとえば、ポリシーに一致する 1 つのイベントが発生すると、アラートが生成され、[通知] ページに表示され、通知が送信されます。 同じポリシーに一致する別のイベントが最初のイベントの 1 分以内に発生した場合、コンプライアンス マネージャーは、新しいアラートをトリガーする代わりに、既存のアラートの [イベント ログ] タブに追加イベントの詳細を追加します。 アラート集約の目的は、アラートの "疲労" を軽減し、より少ないアラートに集中してアクションを実行する方法です。

### <a name="taking-action-on-alerts"></a>アラートに対するアクションの実行

ポリシーの 1 つがアラートを生成すると、アラートの原因となるイベントを表示し、イベントを確認または調査する必要があるかどうかを判断できます。

アラートに対してアクションを実行するには、[アラート] ページの行を選択して、詳細を表示するフライアウト パネルを表示し、[アクション] ボタンを選択し、次に示すオプションの中から選択します。 また、行にカーソルを合わせるとアラート名の左側に表示される丸いボタンを選択し、フィルターの上にあるページの上部にあるアクション ボタンのいずれかを選択することで、アクションを実行することもできます。

**アラートの割** り当て: アラートをユーザーに割り当て、アラートの原因となるイベントを調査または確認できます。 このオプションを選択すると、パネルが開き、組織内のユーザーを選択してアラートを割り当てできます。 [アラート] ページで [フィルター] を選択し、[割り当て済み] フィールドにユーザーの名前を入力することで、アラート **ビューをフィルター処理** できます。

**電子メール** アラート: アラートのアクティビティに関連付けられたユーザーにメールを送信して、アクションが実行されたと確認できます。 このオプションを選択すると、アラートに関する基本情報を含む電子メール テンプレートが開き、詳細な手順をカスタマイズしてユーザーに送信できます。

**ポリシーの詳細を表示** する: アラートをトリガーしたポリシーの設定を確認できます。 このオプションを選択すると、[ポリシーの詳細] パネルが既に開いている  [アラート ポリシー] ページに直接移動します。 ポリシーの詳細パネルを **閉じると** 、[通知] ページに表示されなくなりました。

**状態の変更**: アラートの影響のレビューと調査が必要かどうかに基づいて、アラートの状態を更新できます。 アラートの状態の詳細については、次のセクションをご覧ください。

### <a name="alert-status"></a>アラートの状態

アラートが作成されると、その状態は [アクティブ] **になります**。 各アラートの詳細を確認すると、その状態を以下に示す状態に更新できます。

- **Active**: 状態が変更されるまでのアラートの既定の状態
- **調査中**: アラートが調査中
- **解決済** み: アラートでは、詳細な調査やフォローアップは必要ない
- **[却下**]: アラートが関連しないか、調査を必要としない

アラートの状態を割り当てるか変更するには、表の行からアラートを選択し、フィルターの上にあるページの上部近くの [状態の変更] を選択します。 [アラートの状態の更新] フライアウト ウィンドウで、ドロップダウン メニューから状態を選択し、[アラートの更新] **を選択します**。

アラートが生成されると、その状態は、アラートを生成したポリシーの状態とは独立しています。 たとえば、アクティブなアラートを非アクティブなポリシーに関連付け、その後非アクティブ化または削除されたポリシーによって生成されたアラートに対して調査状態を設定できます。

### <a name="when-policies-are-deleted"></a>ポリシーが削除された場合

ポリシーが削除されると、そのポリシーによって生成されたアラートは [アラート] ページに残りますが、新しいアラートは生成されません。

## <a name="email-notifications-of-alerts"></a>アラートの電子メール通知

ポリシーを作成すると、一致が検出されたという通知をポリシーを作成したユーザーに電子メールが送信されます。 これらの電子メール通知は、組織内の他のユーザーに送信することもできます。 アラートはほぼリアルタイムで発生し、アラートが生成されるとすぐに電子メール通知が送信されます。 電子メールには、イベント名、重大度、検出された時間、コンプライアンス マネージャーでアラートを表示するリンクが含まれる。

### <a name="remove-users-from-receiving-alerts"></a>アラートの受信からユーザーを削除する

アラート受信者を指定し、後で削除する場合は、以下の手順に従います。 ポリシーの一致が検出されると、ポリシー作成者は引き続き電子メール通知を受け取る点に注意してください。

1. ポリシーを編集する [手順を開始します](#edit-a-policy)。
2. [通知の受信者] 画面に到着 **したら** 、[+受信者の選択 **] を選択します**。
3. [受信者の **選択**] フライアウト パネルで、通知から削除するユーザーを見つけて、名前の左側にあるボックスのチェックを外してから、[受信者の追加] ボタン (選択範囲を保存する効果があります) を選択します。
4. ウィザードを続行し、[確認と完了] ページの [受信者] の下にユーザーが表示されないか確認します。 [ **更新] を** 選択して設定を保存し、終了します。
