# <a name="groups-services-interactions"></a>グループ サービスのやり取り

Microsoft 365グループは、エンド ユーザーに接続されたエクスペリエンスを提供するために、Microsoft 365 プラットフォーム内のいくつかのサービスとワークロードに共通のファブリックを提供します。 コアでは、次のMicrosoft 365グループが存在します。

- メンバーシップを管理する方法 (Azure AD)
- メッセージングと会話を行う場所 (メールボックス、Exchange、Microsoft Teams、Yammer)
- ファイルを保存する場所 (SharePoint)
- スケジュールの予定表 (Exchange)
- メモをキャプチャするためのノートブック (OneNote)

グループの作成時点では、他のいくつかのリソースもプロビジョニングされます。ただし、サービスから初めてアクセスするまで表示されません。

- グループ タスクを管理するためのボード (Planner)
- レポート用のワークスペース (Power BI)
- 共有ビデオの領域 (Microsoft Stream)
- 共有フォームの領域 (フォーム)

複数Microsoft 365他のサービスは、グループのメンバーに追加Microsoft 365機能を提供するために、グループグループとやり取りできます。
その例を次に示します。

- Power Appsアプリの場合
- Power Automateの詳細
- Projectに基づくプロジェクト管理の Web とロードマップに関するページ
- Teamsベースの会話の詳細
- Yammerのコミュニティ向けサービス

## <a name="user-interactions-with-groups"></a>グループとのユーザー操作

Microsoft 365グループは、管理者とエンド ユーザーの両方で、さまざまなインターフェイスから作成および管理できます。 

### <a name="administrative-experiences"></a>管理エクスペリエンス

管理者は、いくつかのワークロード管理センター、スクリプトをサポートするコマンド ライン インターフェイス、および Graph API を操作するカスタムビルド アプリから Microsoft 365 グループを作成および管理できます。 この唯一の例外は、Yammerグループです。これは、web インターフェイス内からYammerする必要があります。

**関連する設定**

グループ設定を管理できるさまざまな管理インターフェイスには、注意する必要がある複数の重複が存在します。

**Microsoft 365 管理センター**

このページMicrosoft 365 管理センター、所有者がゲストを追加できる機能と同様に、グループへのゲスト アクセスが既定で有効になっています。 この管理センターからグループに使用できる組織レベルのコントロールはありません。

**Azure AD 管理センター**

管理Azure ADには、ユーザーがグループを作成するか、Azure ポータルで所有者を割り当てるか、有効期限と名前付けポリシーの設定を制御できます。

管理センターには、Microsoft 365 管理センター 以外の所有者がゲストを招待できるかどうかを制限する機能など、Microsoft 365 管理センター 以外のゲストの招待制御手段もいくつか用意されています。

**SharePoint**

SharePointサイトは、所有者、メンバー、および訪問者のセキュリティ グループで作成され、最初の 2 つのセキュリティ グループは、そのグループの対応するMicrosoft 365一致します。 SharePoint Online サイトのメンバーシップは、通常、関連付けられたグループMicrosoft 365管理しますが、双方向の関係ではありません。 Microsoft 365 グループ レベルでのメンバーシップの変更は SharePoint に反映されます。ただし、SharePoint グループでメンバーシップが変更された場合、これは Microsoft 365 グループには反映されません。

### <a name="user-experiences"></a>ユーザー エクスペリエンス

エンド ユーザーは、グループ内のいくつかのサービスからグループをMicrosoft 365、他のユーザーはグループとしか共有できない場合もあります。

グループ作成の制限のエクスペリエンスは、アプリとサービスによって異なります。

## <a name="azure-ad"></a>Azure AD

Azure ADは、すべてのユーザーに基になる ID 管理機能を提供Microsoft 365。

**グループに提供される主な機能**

- グループ メンバーシップ
- 名前付けポリシー
- 有効期限ポリシー
- ゲスト
- グループ作成の制限

**グループAzure AD作成できますか?**

はいMicrosoft 365グループは、管理 web ポータルAzure AD PowerShell 経由、または API から作成Graphできます。

**グループAzure AD存在しませんか?**

はい、Azure ADグループとは関係がない多数のサービスをMicrosoft 365します。 各Microsoft 365グループは、グループ内のオブジェクトとしてAzure AD。

**グループごとに複数のインスタンスAzure ADできますか?**

いいえ、データベースのインスタンスは 1 Azure AD。

**複数Azure ADグループに関連付け可能ですか?**

はい、Azure ADはグループ メンバーシップ サービスを提供する基になるプラットフォームです。

**グループAzure AD関連付けを変更できますか?**

いいえ、Azure ADは、グループが存在する基になるプラットフォームです。

**インスタンスを削除すると、グループが削除されますか?**

グループを削除すると、Azure AD関連するサービスとコンテンツが削除されます。

## <a name="teams"></a>Teams

Teamsは、さまざまな Microsoft およびサード パーティのサービスと対話するための単一のインターフェイスを提供することで、コラボレーションを強化することを目的としたチャット中心のワークスペースです。

既定では、チームが作成されると、Microsoft 365 グループに関連付けられているメールボックスと予定表は、Exchange のグローバル アドレス一覧と Outlook の両方から非表示になります。 これは、ユーザーが同じグループに対してOutlookとTeamsを使用する場合、管理者が手動でMicrosoft 365できます。

**グループに提供される主な機能**

- 会話
- チャネル&タブ
- 会議

**グループTeams作成できますか?**

はい、新しいチームを作成すると、新しいグループMicrosoft 365作成されます。 現在存在しない既存のグループに対してチームを作成できます。

**チームはグループなしで存在しますか?**

いいえ、グループなしでチームが存在することはできません。

**グループごとに複数のチームを作成できますか?**

いいえ、チームとグループの関係は 1:1 です。

**チームを複数のグループに関連付けできますか?**

いいえ、チーム自体は 1 つのグループにのみ関連付けできます。

**チームとグループの関連付けを変更できますか?**

いいえ、チームは、最初に関連付けられたグループにのみ関連付けできます。

**チームを削除すると、グループは削除されますか?**

はい、グループ内のチームMicrosoft Teamsグループ、グループに関連付けられたサービス、およびコンテンツが削除されます。

## <a name="exchange"></a>Exchange

Exchange Online、予定表、連絡先、および関連する機能を提供します。 グループのコンテキストでは、サービス インスタンス全体とは対照的に、1 つのリソースだけが関連付けされます。

**グループに提供される主な機能**

- メールボックスと予定表
- すべてのグループ メンバーにメールを送信する機能
- Storage情報開示Teamsチャネルの会話の詳細、Planner のコメント

**グループExchange作成できますか?**

はい、管理センターからグループを作成Exchange Online、管理センターからグループを作成Outlook。 また、配布リストExchangeグループにMicrosoft 365することもできます。

**グループExchange存在しませんか?**

はい、Exchange Onlineグループの関連付けなしで、共有メールボックスや予定表など、いくつかのサービスを提供します。

**グループごとに複数のメールボックスまたはExchangeインスタンスを作成できますか?**

いいえ、グループのメールボックスと予定表Exchange Online 1 つのみ指定できます。

**メールボックスExchange予定表を複数のグループに関連付けできますか?**

いいえ、メールボックスと予定表には、グループとの 1:1 の関係があります。 メールボックスを他のユーザーまたはグループと共有することもできますが、サービスの関連付けは確立されません。

**メールボックスまたはExchangeとグループの関連付けを変更できますか?**

いいえ、メールボックスと予定表を別のグループに変更することはできません。 ただし、コンテンツは、別のメールボックスから別のメールボックスに移動したり、Outlookツールを使用したりして移動できます。

**メールボックスを削除すると、グループが削除されますか?**

はい、グループ内のメールボックスExchange、グループに関連付けられたサービスとコンテンツが削除されます。

## <a name="forms"></a>Forms

フォームは、Web ベースのアンケートとクイズを提供します。

**グループに提供される主な機能**

- フォームの所有権

**Forms はグループを作成できますか?**

いいえ、Forms はグループを作成できません。

**フォームはグループなしで存在しますか?**

はい、アンケートやクイズはエンド ユーザーのアカウントで直接作成できます。

**グループごとに複数のフォームを作成できますか?**

はい、グループに複数のフォームが関連付けられている場合があります。

**フォームを複数のグループに関連付けできますか?**

いいえ、フォームは 1 つのグループにのみ関連付けできます。

**フォームとグループの関連付けを変更できますか?**

いいえ、フォームがグループに関連付けられたら (直接作成されるか、個人から転送された所有権のいずれか)、別のグループに移動することはできません。

**フォームを削除すると、グループが削除されますか?**

いいえ、Forms インターフェイスからグループを削除できるのは、個々のフォームのみです。

## <a name="onenote"></a>OneNote

OneNoteは、デジタル ノートブック アプリケーションです。 グループOneNoteノートブックは、グループ接続サービスではなく、SharePointサイト内のファイルです。

**グループに提供される主な機能**

- 共有ノートブック (グループに関連付けられたライブラリにSharePoint)

**グループOneNote作成できますか?**

いいえ、アプリケーションOneNoteグループを作成できません。

**グループOneNoteノートブックは存在しませんか?**

はい、ノートブックは、ユーザーまたは他の共有OneDriveに直接作成できます。

**グループごとに複数のOneNoteノートブックを使用できますか?**

はい、既定ではノートブックが作成され、他のユーザーも追加できます。ただし、グループ関連サービスから OneNote へのリンクは常に既定のノートブックに移動します。

**ノートブックをOneNoteグループに関連付けできますか?**

いいえ、ノートブックはグループに関連付けられたサイト ライブラリSharePointに格納され、さまざまなインターフェイスからリンクされます。 ただし、個人と共有できるのと同じ方法で他のグループと共有することもできます。

**ノートブックとグループの関連付けを変更できますか?**

いいえ、ノートブック自体はグループに関連付けられているので、他のグループ接続サービスから直接アクセスすることもできますが、コンテンツは OneNote アプリケーション内でノートブック間で移動できます。

**ノートブックを削除すると、グループが削除されますか?**

ただし、ノートブックが削除OneNote、グループに関連付けられたサービスの一部にリンクが壊れている可能性があります。

## <a name="planner"></a>Planner

Planner は、軽量なグループ タスク管理サービスです。

**グループに提供される主な機能**

- グループ タスクを管理するためのボード

**Planner はグループを作成できますか?**

はい、プランの作成によって新しいグループが作成されます。

**Planner ボードはグループなしで存在しますか?**

いいえ、プランはグループに関連付けられている必要があります。

**グループごとに複数のプランを作成できますか?**

はい、グループごとに複数のプランを作成できます。

**プランを複数のグループに関連付けできますか?**

いいえ、プランはグループ メンバーシップのみに依存してアクセスを決定します。

**プランとグループの関連付けを変更できますか?**

ただし、プランをコピーすると新しいグループが作成されます。

> [!NOTE]
> 他のアプリケーションによって作成されたグループは、ユーザーの Planner に自動的に表示されません。 最初にボードにアクセスするには、グループ ベースの別のインターフェイス (Outlook など) からボードを開く必要があります。

**プランを削除すると、グループが削除されますか?**

はい、プランを削除すると、グループとグループに関連付けられたサービスとコンテンツが削除されます。

## <a name="power-apps"></a>Power Apps

**グループに提供される主な機能**

- アプリは、実行および変更するグループと共有できます

はい、アプリは、共有または公開されるまで、Power Apps内に作成し、作成者アカウント内に存在できます。

**グループごとに複数のアプリを使用できますか?**

はい、複数のアプリがグループと共有されている場合があります。

**アプリを複数のグループに関連付けできますか?**

はい、アプリは複数のグループと共有できます。

**アプリとグループの関連付けを変更できますか?**

## <a name="power-automate"></a>Power Automate

Power Automate (以前は Microsoft Flow) は、ワークフローとオートメーション サービスを提供します。

**グループに提供される主な機能**

- ワークフローは、実行および変更するグループと共有できます。

ただし、セキュリティ グループの作成やグループのメンバーシップの更新など、Azure ADを実行するフローをMicrosoft 365できます。

**フローはグループなしで存在しますか?**

はい、フローは、共有または公開されるまで、Power Automate内に作成し、作成者アカウント内に存在できます。

**グループごとに複数のフローを使用できますか?**

はい、複数のフローがグループと共有されている場合があります。

**フローを複数のグループに関連付けできますか?**

はい、フローは複数のグループと共有できます。

**フローとグループの関連付けを変更できますか?**

はい、Power Automateグループと Microsoft 365の関連付けは共有のみ行うので、フローは作成者と引き続き存在します。

**フローを削除すると、グループは削除されますか?**

いいえ、Power Appsと同様に、フローはグループと共有される以外にグループに接続されていません。

Power BIは、対話型のデータ駆動型ダッシュボードとレポートを提供します。

**グループに提供される主な機能**

- データレポート

**グループPower BI作成できますか?**

はい、従来のワークスペースを作成すると、グループMicrosoft 365作成されます。

**クラシック ワークスペースPower BIグループが存在しない場合**

いいえ、[グループ内のクラシック ワークスペースPower BIグループに関連付けられている必要があります](/power-bi/collaborate-share/service-collaborate-power-bi-workspace)。
