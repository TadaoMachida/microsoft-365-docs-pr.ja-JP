# <a name="plan-external-collaboration"></a>外部コラボレーションを計画する

Microsoft 365、組織外のユーザーと共同作業するためのいくつかのオプションが用意されています。

- 組織外のユーザーと 1:1 Teamsチャット
- Teams外のユーザーとの会議の開催
- 組織外のユーザーと個々のファイルまたはフォルダーを共有する
- チャネルの会話、ファイルの共同作業、共有アプリを使用したチームでの共同作業

この記事では、4 つ目のオプション、チャネル会話とのグループ コラボレーション、ファイルの共同作業、共有アプリについて説明します。 

## <a name="terms"></a>Terms

- **Azure AD B2B コラボレーション** – ユーザーが組織外のユーザーとファイル、フォルダー、サイト、グループ、チームを共有できる機能。 これらのユーザーは、ディレクトリ内のゲスト アカウントを使用して共有リソースにアクセスします。
- **Azure AD B2B** ダイレクト 接続 – ユーザーが組織内のリソースを他の組織のユーザーと共有Azure ADします。 これらのユーザーは、自分の仕事または学校のアカウントを使用して共有リソースにアクセスします。 組織にゲスト アカウントは作成されません。
- **外部参加者** – ディレクトリ内のゲスト アカウントではなく、自分の ID を使用してリソース (共有チャネルなど) に参加している組織外のユーザー。
- **外部組織** – リソースを共有している別の組織。
- **ゲスト** – ディレクトリ内のゲスト アカウントにサインインして共有リソースにアクセスする組織外のユーザー。
- **ホスト組織** – 共有リソース (共有チャネルなど) をホストしている組織。
- **共有チャネル** – チームTeams人と共有できるチャネルの 1 つ。 これらのユーザーは、組織内または他の組織Azure ADできます。
- **リンクの** 共有 – そのファイルまたはフォルダーを他のユーザーと共有するために使用されるファイルまたはフォルダーへのアクセス許可を持つリンク。 共有されているユーザーは、組織の内部または外部にできます。

## <a name="options-for-collaboration-in-a-team"></a>チームでの共同作業のオプション

組織外のユーザーとチームで共同作業を行う場合、それらのユーザーが共有するリソースにアクセスする方法には 2 つのオプションがあります。

**ゲスト共有**

ゲスト共有では、Azure AD B2B コラボレーションを使用して、ユーザーごとにゲスト アカウントを追加することで、組織外のAzure AD共有および共同作業を許可します。 ゲスト アカウントは、次の場合に使用できます。

- Teams、SharePoint、Microsoft 365グループのゲスト メンバーシップ
- 個々のファイルとフォルダーの共有

チーム内のゲストは、通常のチーム メンバーと同様の機能を持っています。

**共有チャネルの外部参加者**

外部参加者は、独自のユーザー ID またはユーザー ID を使用してAzure AD共有Microsoft 365アクセスします。 これは、両方の組織Azure AD構成された組織関係を介して B2B ダイレクト接続を使用して有効になります。 ゲスト アカウントは、このリレーションシップでは使用されません。

共有チャネルとゲスト共有の外部参加者の主な利点は、組織外のユーザーがユーザー コンテキストを変更することなく、Teams でユーザーと共同作業を行えるという利点です。 ゲスト アカウントを使用する場合、ユーザーは自分のTeamsまたは学校のアカウントでアカウントからサインアウトし、ゲスト アカウントを使用してもう一度サインインする必要があります。 または、プライベート ブラウザー セッションで実行されているTeamsコピーを使用することもできます。 この組織間の切り替えには時間がかかるので、ユーザーが特定の組織からサインアウトしている間に重要な通信を見逃す可能性があります。

共有チャネルを使用すると、ユーザーは自分の組織にサインインしたままで、他の組織から共有されているチャネルにアクセスできます。 他の組織の共有チャネルは、Teamsチームとチャネルと共に利用できます。 組織を切り替える必要はありません。

## <a name="planning-considerations"></a>計画に関する考慮事項

ほとんどの組織では、外部参加者とゲスト共有チャネルと共有チャネルの両方を使用します。 

ゲスト共有は、Azure AD および Microsoft 365 (Teams、Microsoft 365、および SharePoint) で有効になっています。 これにより、ユーザーは IT の支援を要求することなく、チームやサイトにゲストを招待し、ファイルを共有できます。

次の場合は、ゲスト共有を使用する必要があります。
- 組織外のユーザーを個々のチャネルではなくチームに招待する場合
- チャネル内のファイルまたはフォルダーを、チャネルに所属していない組織外のユーザーと共有する場合
- 仕事用または学校用のアカウントを持つ組織外のユーザーと共同作業を行います。

共有チャネルは Teams で既定で有効になっていますが、共有チャネルとの外部コラボレーションでは、Azure AD 管理者が組織と共有する組織間のクロステナント アクセスをセットアップする必要があります。 他の組織は、テナント間アクセスをエンドで設定する必要があります。

他の組織と共有チャネルを使用する場合は、セルフサービス モデルと要求によるモデルの間で選択できます。

- セルフサービス – テナント間アクセスを構成して、他のすべての組織への受信および送信アクセスを許可Azure ADできます。 または、ユーザーが共有しない組織の一覧をブロックして、他のすべての組織を使用できます。 これにより、ユーザーは、ヘルプデスクや IT 部門に問い合わせることなく、組織外のユーザーを招待して共有チャネルに参加できます。
- 要求による – 共有チャネルを許可する個々の組織ごとに、テナント間アクセスを構成できます。 このモデルを選択する場合は、ドキュメント化されたビジネス プロセスをユーザーがフォローして、別の組織とのテナント間アクセスを要求することが重要です。

## <a name="compliance-in-shared-channels"></a>共有チャネルのコンプライアンス

共有チャネルは、コンプライアンス機能Microsoft 365統合されています。

##### <a name="communications-compliance"></a>コミュニケーションコンプライアンス

管理者は、チャネル内のすべてのユーザーのコンテンツを監視するポリシーを設定できます。 チャネル内のすべてのメッセージ コンテンツ (共有チャネルを含む) は、通信コンプライアンス [ポリシーによってカバーされます](/microsoft-365/compliance/communication-compliance)。 共有チャネルは、ホスト組織のポリシーを継承します。

##### <a name="conditional-access"></a>条件付きアクセス

ホスト組織の条件付きアクセス [ポリシー](/azure/active-directory/conditional-access/overview) は、B2B ダイレクト接続ユーザーを含む外部参加者に適用されます。 外部組織のポリシーは使用されません。 共有チャネルでは、次の種類の条件付きアクセス ポリシーがサポートされています。

- すべてのゲスト ユーザー、外部参加者、オンライン クラウド アプリを対象SharePointポリシー
- MFA、準拠デバイス、またはハイブリッド デバイスが参加しているデバイスを必要とするアクセスAzure AD付与します。 

IP ベースのポリシーは、ファイル レベルでSharePointされます。 そのため、外部参加者は制限された場所から共有チャネルにアクセスできますが、ファイルを開く際にブロックされます。

##### <a name="data-loss-prevention-dlp"></a>データ損失防止 (DLP)

管理者は、 [共有チャネルを](/microsoft-365/compliance/dlp-policy-design) 含むすべてのチャネルがポリシーを継承するチームに DLP ポリシーを適用できます。 共有チャネルは、ホスト組織のポリシーを継承します。

##### <a name="retention-policy"></a>アイテム保持ポリシー

管理者は、 [共有チャネルを](/microsoft-365/compliance/retention) 含むすべてのチャネルが保持ポリシーを継承するチームに保持ポリシーを適用できます。 共有チャネルは、親チームのポリシーを継承します。

##### <a name="sensitivity-labels"></a>秘密度ラベル

[ホスト組織で](/microsoft-365/compliance/sensitivity-labels) 使用できる感度ラベルは、共有チャネル サイト内のドキュメントに適用できる唯一のラベルです。 感度ラベルで暗号化されたファイルは、外部参加者が開くことができません。 自動ラベル付けは使用されません。

共有チャネルと関連付SharePointサイトは、親チームからラベルを継承します。

##### <a name="information-barriers"></a>情報バリア

情報バリア ポリシーごとに通信を許可されていないユーザー[](/microsoftteams/information-barriers-in-teams)は、共有チャネルに参加できない。 情報バリア ポリシーは、ホスト組織内のユーザーにのみ有効です。 ユーザーが別の組織の共有チャネルの外部参加者である場合、情報バリア ポリシーは適用されません。

##### <a name="ediscovery"></a>電子情報開示

管理者は、チャネル内のすべてのユーザーに対して検索を実行できます。 共有チャネルを含むすべてのチャネルが検出可能です。 データを追加したユーザーに関係なく、チャネル内のすべてのメッセージ データは、コンプライアンス管理者が検出できます。

##### <a name="legal-hold"></a>訴訟ホールド

管理者は、チームの一部ではないホスト組織のチャネル専用メンバーを保留にできます。 チーム全体 [を保留に設定できます](/MicrosoftTeams/legal-hold)。 管理者は、外部参加者を保留にすることはできません。

##### <a name="audit-logs"></a>監査ログ

既存の監査イベントに [対して実行されるアクションはすべて](/microsoft-365/compliance/detailed-properties-in-the-office-365-audit-log) 、共有チャネルで監査されます。
